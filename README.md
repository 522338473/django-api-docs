# django-api

## 前言

Django-REST-Framework(简称drf)应该是现在, 使用Django的Python程序员的API首选框架, 但是它是否应该是首选的真的值得深思熟虑

drf一旦你的项目真的使用起来了, 我觉得你的麻烦也不远了, 因为他真的不适合我们在工作中写API

So, 这个仓库暂时先预留, 虽然我已经在我们公司的项目中使用了, 但是也是应急, 待完善后再PUSH上来, 额...这个时间应该不会太久

## 介绍

接口应该是这个样子的:

1. 我们并不是所有的接口都必须遵循REST规范, 它只是一种建议

2. 数据库的字段也并不一定是API所需要的参数, 数据库的字段也不一定是API返回的数据

3. 接口文档应该算作API的一部分

4. 写接口应该优雅而迅速

5. 不管是什么项目, 它应该是灵活的
